# 관리자 알림톡 운영 가이드

## 목적

`/admin/alimtalk` 화면은 펫매니저 내부 운영자가 알림톡 연동 상태를 확인하고, 쏘다 템플릿을 등록/검수 요청하고, 승인된 템플릿 코드를 릴레이 서버에 연결하는 화면입니다.

가장 중요한 현재 목적은 **반려동물 미용 전후 사진 알림톡 템플릿을 쏘다에 등록하고 카카오 검수 요청까지 보내는 것**입니다.

## 어디서 하나요

1. 운영 사이트에 접속합니다.
   - `https://www.petmanager.co.kr/admin`
2. 관리자 계정으로 로그인합니다.
3. `알림톡 키 / 템플릿 관리` 또는 `/admin/alimtalk` 화면으로 이동합니다.
4. 아래 순서로 확인합니다.

## 화면별 의미

### 1. 실시간 릴레이 진단

릴레이 서버가 정상인지 확인하는 영역입니다.

- `/health`가 `200`이면 릴레이 서버가 살아 있습니다.
- `/debug/templates`가 정상 응답하면 템플릿 매핑 상태를 읽을 수 있습니다.
- 여기서 실패하면 템플릿 등록이나 실제 발송도 실패할 가능성이 큽니다.

### 2. 연결값 / 템플릿 코드 수정

릴레이 서버가 쏘다 API를 호출할 때 쓰는 값입니다.

- `Relay Secret`
  - Vercel의 `ALIMTALK_RELAY_SECRET`과 같아야 합니다.
  - 앱 서버가 릴레이 서버를 호출할 때 쓰는 내부 인증키입니다.
- `쏘다 발송 URL`
  - 실제 알림톡 발송 API 주소입니다.
- `쏘다 발송 조회 URL`
  - 발송 결과를 조회할 때 쓰는 API 주소입니다.
- `쏘다 API Key`
  - 쏘다 API 인증키입니다.
- `쏘다 Token Key`
  - 쏘다 계정/서비스 토큰입니다.
- `쏘다 Sender Key`
  - 카카오 발신 프로필에 연결된 Sender Key입니다.
- `릴레이 템플릿 코드 매핑`
  - 예약 확정, 예약 취소, 미용 완료 같은 펫매니저 알림 종류와 쏘다 템플릿 코드를 연결합니다.
  - 승인된 템플릿 코드만 넣어야 실제 발송됩니다.

주의: 연결 원본값은 자주 수정하는 영역이 아닙니다. 보통 운영 초기 세팅 후에는 템플릿 코드만 관리합니다.

### 3. 앱 서버 현재 읽히는 환경변수

Vercel 서버가 현재 읽고 있는 값을 보여주는 확인용 영역입니다.

- 이 영역은 읽기 전용입니다.
- 여기서 값을 입력해도 Vercel 환경변수가 바뀌는 것은 아닙니다.
- 실제 운영 발송은 현재 구조상 릴레이 서버 설정과 템플릿 매핑을 기준으로 봅니다.

### 4. 전후 사진 알림톡 템플릿 만들기

쏘다 템플릿 등록과 검수 요청을 보내는 영역입니다.

기본 순서:

1. `전후 사진 기본값`을 눌러 기본 문구를 불러옵니다.
2. `템플릿 코드`를 확인합니다.
   - 예: `grooming_completed_photo`
   - 영문, 숫자, `_`, `-`만 사용합니다.
3. `템플릿 이름`을 확인합니다.
   - 예: `미용 전후 사진 안내`
4. `카테고리`를 선택합니다.
   - 쏘다에서 내려주는 카카오 템플릿 카테고리입니다.
5. `등록 후 릴레이 매핑`을 확인합니다.
   - 전후 사진용이면 `미용 완료/전후 사진`을 선택합니다.
6. `템플릿 본문`을 확인합니다.
7. `등록 후 바로 검수 요청`이 켜져 있는지 확인합니다.
8. `코드 검증`을 누릅니다.
9. 문제가 없으면 `템플릿 등록/검수 요청`을 누릅니다.

정상 처리되면 쏘다에 템플릿이 등록되고, 카카오 검수 요청 단계로 넘어갑니다.

## 전후 사진용 기본 문구

현재 기본 문구는 아래 기준입니다.

```text
[#{매장명}]
#{반려동물명} 미용 전후 사진을 보내드려요.

사진은 개인정보 보호를 위해 일정 기간 후 자동 보관 정책에 따라 정리될 수 있습니다.
궁금한 점이 있으면 매장으로 문의해 주세요.
```

중요: 카카오 알림톡은 승인된 템플릿 본문과 실제 발송 본문이 다르면 발송이 거절될 수 있습니다. 템플릿을 바꾸면 실제 발송 문구도 같은 기준으로 맞춰야 합니다.

## 승인 전/후 상태

템플릿 등록 버튼을 누른다고 바로 고객에게 발송 가능한 것은 아닙니다.

- 등록 성공
  - 쏘다에 템플릿 등록 요청이 성공한 상태입니다.
- 검수 요청 성공
  - 카카오 검수 대기로 넘어간 상태입니다.
- 승인 완료
  - 실제 고객 발송에 사용할 수 있는 상태입니다.
- 반려/실패
  - 문구, 카테고리, 변수, 광고성 여부 등을 수정해야 합니다.

승인 전에는 오너 화면에서 전후 사진 알림톡을 눌러도 실제 발송하지 않고, 요청만 저장하거나 실패 상태로 남기는 것이 정상입니다.

## 실제 발송 테스트 순서

1. 쏘다/카카오에서 템플릿 승인 완료를 확인합니다.
2. `/admin/alimtalk`의 템플릿 비교 영역에서 `미용 완료/전후 사진` 코드와 상태를 확인합니다.
3. 오너 화면에서 예약 상세를 엽니다.
4. 전/후 사진을 업로드합니다.
5. `전후 사진 알림톡 보내기`를 누릅니다.
6. 고객 휴대폰으로 알림톡이 도착하는지 확인합니다.
7. DB의 알림/미디어 첨부 기록이 남는지 확인합니다.

## 문제가 생겼을 때 보는 순서

1. 릴레이 진단 `/health`가 200인지 확인합니다.
2. Vercel 환경변수 `ALIMTALK_RELAY_URL`, `ALIMTALK_RELAY_SECRET`이 있는지 확인합니다.
3. 릴레이 설정의 `Relay Secret`이 Vercel의 `ALIMTALK_RELAY_SECRET`과 같은지 확인합니다.
4. 쏘다 API Key, Token Key, Sender Key가 맞는지 확인합니다.
5. 템플릿 코드가 승인 완료된 코드인지 확인합니다.
6. 실제 발송 문구가 승인된 템플릿 본문과 같은지 확인합니다.
