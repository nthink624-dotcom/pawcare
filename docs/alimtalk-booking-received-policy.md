# 예약 접수 알림톡 운영 기준

## 결론

현재 PetManager에서는 `예약 접수` 고객 알림톡을 기본 발송하지 않습니다.

## 뜻

고객이 예약을 등록하면 고객 화면에서 이미 예약 접수 완료 내용을 바로 확인합니다. 그래서 같은 순간에 알림톡까지 보내면 고객 입장에서는 같은 안내를 두 번 받는 느낌이 날 수 있습니다.

## 빼는 이유

- 예약 완료 화면에서 이미 접수 사실을 보여줍니다.
- 매장이 아직 승인하지 않은 예약을 알림톡으로 다시 보내면 확정처럼 느껴질 수 있습니다.
- 알림톡 건수를 아낄 수 있습니다.
- 고객에게 중요한 알림은 `예약 확정`, `예약 거절`, `예약 변경`, `방문 전 알림`, `미용 시작`, `픽업 준비`, `미용 완료` 쪽입니다.

## 다시 넣는 게 좋은 경우

- 고객이 예약 완료 화면을 닫아도 접수 내용을 카카오톡으로 남겨야 할 때
- 예약 접수 후 고객이 예약 확인 링크를 다시 찾아야 하는 경우가 많을 때
- 매장 운영자가 “접수만 되어도 고객에게 반드시 카카오톡을 보내고 싶다”고 요구할 때

## 현재 동작

- 고객 예약 등록 시 `예약 접수` 알림톡은 발송하지 않습니다.
- 오너에게 보이는 `새 예약이 접수되었어요` 인앱 알림은 유지합니다.
- 알림톡 관페의 템플릿 비교 목록에서도 `예약 접수`는 숨깁니다.
