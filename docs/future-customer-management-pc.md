# Future Customer Management PC

## 1. 목적

- 현재 고객관리는 오너 모바일 앱 기준으로 운영한다.
- 향후 PC 버전은 별도 화면군으로 설계한다.
- 모바일 고객관리 화면을 단순 확대해서 PC 버전을 만들지 않는다.

---

## 2. 현재 고객관리 기준

- 현재 고객관리는 `오너 모바일 앱` 화면군에 속한다.
- 대상은 오너가 휴대폰에서 고객 목록을 조회하고, 빠르게 확인하고, 짧은 수정과 연락 액션을 처리하는 흐름이다.
- 현재 기준 문법:
  - `max-width: 430px`
  - 하단 탭 구조
  - 모바일 리스트 row 또는 카드형 목록
  - 모바일 바텀시트
  - 모바일 full-width CTA

이 문법은 PC에 그대로 확장하면 안 된다.

---

## 3. 모바일 UI를 단순 확대하지 않는 이유

### 이유 1. 정보 밀도 목적이 다르다

- 모바일 고객관리는 빠른 조회와 짧은 액션이 목적이다.
- PC 고객관리는 많은 고객을 한 번에 비교하고, 필터링하고, 상세를 오래 보는 작업이 목적이다.

### 이유 2. 화면 구조가 다르다

- 모바일은 한 번에 한 작업에 집중해야 한다.
- PC는 목록과 상세를 동시에 보여주는 구조가 더 효율적이다.

### 이유 3. 액션 위치가 다르다

- 모바일은 하단 CTA가 효율적이다.
- PC는 상단 액션 바, 행 액션, 상세 패널 하단 액션이 더 적합하다.

### 이유 4. 바텀시트 문법이 PC에 맞지 않는다

- 모바일 바텀시트는 손가락 터치와 짧은 선택에 맞춰진 문법이다.
- PC에서는 바텀시트보다 우측 상세 패널 또는 별도 상세 페이지가 더 명확하다.

### 이유 5. 카드형 리스트를 늘리는 방식은 PC에서 비효율적이다

- 모바일 카드형 리스트를 단순 확대하면 한 화면에 보여줄 수 있는 고객 수가 적고, 스캔 속도가 느리다.
- PC에서는 row 또는 table 구조가 더 적합하다.

---

## 4. PC 고객관리에서 필요한 구조

향후 PC 고객관리 화면은 아래 구조를 기본으로 설계한다.

### 검색

- 고객명
- 연락처
- 반려동물명
- 메모 키워드

### 필터

- 활성/비활성 고객
- 최근 방문 여부
- 알림 수신 여부
- 태그/메모 상태

### 고객 목록

- row 또는 table 기반
- 기본 컬럼:
  - 고객명
  - 연락처
  - 반려동물 수
  - 최근 방문일
  - 예약 예정 여부
  - 상태

### 고객 상세 패널

- 우측 패널 또는 별도 상세 페이지
- 목록 선택 상태와 연결

### 반려동물 정보

- 이름
- 품종
- 나이/생년 정보
- 특징/특이사항

### 방문/미용 기록

- 최근 미용 이력
- 서비스 종류
- 담당자/메모

### 예약 이력

- 예정 예약
- 완료 예약
- 취소/변경 기록

### 메모

- 고객 메모
- 주의사항
- 내부 운영 메모

### 일괄 관리

- 태그 변경
- 알림 설정 변경
- 삭제/복구
- 상태 변경

---

## 5. 모바일 버전에서 재사용 가능한 것

다음 요소는 모바일 고객관리와 PC 고객관리에서 공통 기준으로 재사용한다.

### 상태 배지

- 상태 문구
- 상태 색상 역할
- 상태 우선순위

### 고객 정보 위계

- 고객명
- 연락처
- 반려동물 정보
- 메타 정보

### EmptyState 문구

- 고객 없음
- 검색 결과 없음
- 예약 이력 없음

### 기본 버튼 의미

- 저장
- 취소
- 삭제
- 복구
- 편집

버튼의 의미는 공통으로 유지하되, 위치와 레이아웃은 PC 기준으로 다시 설계한다.

---

## 6. 재사용하면 안 되는 것

다음 요소는 모바일 전용 문법이므로 PC 고객관리 화면에 그대로 가져오지 않는다.

### 모바일 하단 탭

- PC에서는 좌측 내비게이션, 상단 탭, 목록/상세 구조를 사용한다.

### 모바일 바텀시트

- PC에서는 우측 상세 패널 또는 다이얼로그를 사용한다.

### 모바일 full-width CTA

- PC에서는 상단 액션 바, 상세 패널 하단 액션, 행 액션으로 재배치한다.

### 모바일 카드형 리스트를 그대로 늘린 구조

- PC에서는 row 또는 table 기반으로 재설계한다.

---

## 7. 권장 레이아웃 구조

향후 PC 고객관리 기본 구조:

1. 상단 헤더
2. 검색/필터 바
3. 고객 목록 영역
4. 우측 고객 상세 패널

이 구조를 기본으로 두고, 목록과 상세를 동시에 보면서 작업할 수 있게 설계한다.

---

## 8. 결론

- 현재 고객관리는 오너 모바일 앱 기준으로 유지한다.
- 향후 PC 버전은 별도 화면군으로 설계한다.
- 모바일과 PC는 상태 문구, 색상 토큰, 타이포 위계는 최대한 공유한다.
- 하지만 레이아웃, CTA 위치, 레이어 구조, 목록 표현 방식은 분리한다.
