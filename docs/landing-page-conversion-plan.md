# 넘친 Day 랜딩페이지 전환 기획서

## 1. 목적

- 1차 전환: 미용샵 오너가 `14일 무료 시작`을 눌러 가입을 완료한다.
- 2차 전환: 바로 가입하지 않는 방문자가 실제 제품 화면과 요금제를 확인한다.
- 핵심 인식: 넘친 Day는 단순 캘린더가 아니라, 미용 중 놓치기 쉬운 예약과 고객 기록을 대신 이어 주는 반려동물 미용샵 운영 도구다.
- 랜딩페이지의 역할은 기능을 많이 보여주는 것이 아니라 `내 문제다 -> 실제로 해결된다 -> 가격보다 이득이다 -> 일단 써보자`의 판단을 빠르게 만드는 것이다.

## 2. 핵심 고객

### 1차 고객

- 미용을 직접 하면서 전화, 문자, DM 예약까지 함께 받는 1인 미용샵 오너
- 메모장, 카카오톡, 캘린더에 예약과 고객 정보를 나누어 기록하는 오너
- 보호자 연락처, 반려동물 정보, 이전 방문 내용을 반복해서 찾거나 다시 묻는 오너

### 2차 고객

- 직원별 예약을 한 화면에서 나누어 봐야 하는 2~4인 매장
- 예약량과 알림톡 발송량이 많고 직원 권한이 필요한 5인 이상 매장
- 동일 사업자로 복수 업장을 운영하며 별도 안내가 필요한 오너

## 3. 구매를 만드는 핵심 메시지

### 메인 포지셔닝

`반려동물 미용샵 예약·고객관리, 넘친 Day`

### 핵심 설명

`미용 중 전화를 못 받아도 고객은 직접 예약하고, 보호자·연락처·반려동물 정보는 고객장부에 자동으로 쌓입니다.`

### 가장 먼저 증명할 가치

1. 놓치던 예약을 고객이 직접 남긴다.
2. 예약 입력이 곧 고객정보 등록이 된다.
3. 오늘 예약, 직원 일정, 고객 기록을 한 화면에서 본다.
4. 방문 안내 알림톡을 직접 챙기는 시간을 줄인다.
5. 하루 30분을 줄이면 월 15시간, 최저시급 기준 154,800원 상당의 시간을 확보할 수 있다.

절약액은 반드시 `업무시간 가치 환산 예시이며 실제 절감액을 보장하지 않음`을 함께 표시한다.

## 4. 권장 구조: 총 9개 섹션

헤더와 푸터는 섹션 수에서 제외한다. 9개보다 늘리면 같은 기능을 반복 설명하게 되고, 7개 이하로 줄이면 실제 제품 신뢰와 가격 불안을 충분히 해소하기 어렵다.

### 섹션 1. 히어로: 3초 안에 누구를 위한 무엇인지 전달

- H1: `반려동물 미용샵 예약·고객관리, 넘친 Day`
- 보조 문구: `미용 중 전화를 못 받아도 고객은 직접 예약하고, 보호자·반려동물 정보는 고객장부에 자동으로 쌓입니다.`
- 기본 CTA: `14일 무료로 시작하기`
- 보조 CTA: `실제 화면 먼저 보기`
- 안심 문구: `카드 등록 없이 시작 · PC와 모바일에서 확인`
- 비주얼: 실제 오너 예약관리 화면을 전체 히어로 배경으로 크게 사용한다. 텍스트와 이미지를 두 칸으로 나눈 홍보형 구성을 피한다.
- 다음 섹션의 첫 문장이 화면 하단에 일부 보이도록 높이를 제한한다.

전환 역할: 방문자가 자신을 대상 고객으로 즉시 인식하고 첫 행동을 선택하게 한다.

### 섹션 2. 문제 인식: 바쁜 것이 아니라 운영 시간이 새고 있음을 보여주기

- 제목: `미용하는 동안, 예약과 고객 기록은 계속 쌓입니다`
- 세 가지 실제 상황만 사용한다.
  - 미용 중 전화가 와도 받을 수 없다.
  - 문자와 DM의 예약 내용을 다시 캘린더에 옮긴다.
  - 보호자 연락처와 반려동물 정보를 다음 방문 때 다시 찾는다.
- 카드 4개 나열 대신 한 줄 업무 흐름 또는 전·후 비교로 구성한다.
- 하단에 `하루 30분 -> 월 15시간`을 작은 증거 막대로 먼저 노출한다.

전환 역할: 오너가 이미 감수하던 반복 업무를 비용으로 인식하게 한다.

### 섹션 3. 핵심 차별점: 예약 하나가 고객 DB가 되는 과정

- 제목: `고객이 예약하면, 고객정보 등록까지 끝납니다`
- 실제 화면 세 개를 하나의 흐름으로 연결한다.
  1. 고객이 예약페이지에서 보호자명, 연락처, 반려동물 정보를 입력한다.
  2. 예약이 오너 화면에 들어온다.
  3. 해당 정보가 고객관리 화면에 자동으로 이어진다.
- 사용 이미지: 고객 예약 입력 화면, 예약 접수 화면, 오너 고객관리 화면의 최신 실제 캡처
- 설명 문구는 데이터 필드명을 나열하기보다 `다음 예약 때 다시 묻지 않아도 됩니다`로 마무리한다.

전환 역할: 일반 캘린더와 다른 넘친 Day만의 필요성을 만든다.

### 섹션 4. 실제 운영 화면: 오늘 할 일을 한 화면에서 판단

- 제목: `오늘 예약과 직원별 일정을 열자마자 확인하세요`
- 실제 예약관리 화면을 크게 한 장 보여준다.
- 화면 위에 가짜 목업 설명을 덧씌우지 않고, 실제 UI의 세 지점만 번호로 연결한다.
  - 직원별 예약 열
  - 예약 상태와 시간
  - 고객·반려동물 상세 정보
- 보조로 월간 캘린더 화면을 한 장 사용한다.

전환 역할: 제품이 실제로 존재하고 반복 업무에 사용할 수 있다는 신뢰를 준다.

### 섹션 5. 자동화: 예약을 받은 뒤 해야 할 일까지 줄이기

- 제목: `예약을 받은 뒤의 안내도 매번 직접 챙기지 마세요`
- 흐름: `예약 접수 -> 방문 안내 -> 미용 진행 -> 완료 안내`
- 방문 안내 알림톡은 승인된 정책대로 예약당 가장 적절한 시점에 1회 자동 발송된다는 범위만 설명한다.
- 알림 설정과 발송 현황의 실제 캡처를 사용한다.
- 포함 알림톡 소진 후 추가 충전이 필요하다는 사실은 숨기지 않고 요금 섹션과 연결한다.

전환 역할: 예약관리 프로그램이 아니라 실제 노동을 줄이는 도구로 인식시킨다.

### 섹션 6. 시간 가치: 가격보다 먼저 경제성을 이해시키기

- 제목: `하루 30분을 되찾으면, 한 달에 15시간입니다`
- 계산식: `0.5시간 x 30일 = 15시간`
- 2026년 최저임금 기준: `15시간 x 10,320원 = 154,800원`
- 비교 문구: `1인 운영 플랜 월 19,000원부터`
- 표현 원칙: `월 154,800원 절약 보장`이 아니라 `업무시간을 최저시급으로 환산하면 월 154,800원 상당`으로 쓴다.
- 숫자 세 개만 크게 보여주고 장식용 카드 반복은 피한다.

전환 역할: 월 구독료를 비용이 아니라 시간 회수 비용으로 재해석한다.

### 섹션 7. 도입 불안 해소: 실제 제품과 시작 조건을 투명하게

- 제목: `만들어 낸 목업 없이, 지금 사용하는 화면만 보여드립니다`
- 포함 항목:
  - 랜딩의 모든 제품 이미지는 실제 서비스 캡처
  - 14일 무료체험
  - 카드 등록 없이 시작
  - 운영 인원에 맞춰 플랜 선택
  - 이용 중 정기결제 취소 가능
- 실제 고객 후기와 사용 수치는 확보 전까지 넣지 않는다.
- 향후 실제 후기를 확보하면 이 위치에 `매장명·운영 인원·사용 기간이 확인된 후기`만 추가한다.

전환 역할: 신생 서비스에 대한 과장·결제·해지 불안을 줄인다.

### 섹션 8. 요금제: 기능표보다 내 매장에 맞는 선택을 쉽게

- 제목: `운영 인원에 맞는 플랜만 고르세요`
- 1인 운영: 월 19,000원, 알림톡 500건
- 2~4인 운영: 월 29,000원, 알림톡 1,500건
- 5인 이상 운영: 월 79,000원, 알림톡 5,000건
- 공통 기능은 카드마다 반복하지 않고 요금제 상단 한 줄로 묶는다.
- 각 카드의 CTA는 `이 플랜으로 14일 시작`으로 구체화한다.
- 다점포 할인은 상세 금액 대신 현재 정책을 짧게 안내하고 별도 문의 흐름을 둔다.
- 초과 알림톡 11원/건, 부가세 포함을 가격 아래에서 바로 확인할 수 있게 한다.

전환 역할: 비교 피로를 줄이고 가입 버튼을 누를 명확한 이유를 준다.

### 섹션 9. FAQ + 최종 CTA: 마지막 반대 이유 제거

- FAQ는 6개 이내로 제한한다.
  1. 고객도 앱을 설치해야 하나요?
  2. 보호자와 반려동물 정보는 어떻게 저장되나요?
  3. 직원별 예약을 나누어 볼 수 있나요?
  4. 알림톡은 언제 자동으로 발송되나요?
  5. 플랜을 바꾸거나 정기결제를 취소할 수 있나요?
  6. 여러 매장을 운영하면 어떻게 하나요?
- 답변은 실제 구현과 정책 파일에서 확인된 내용만 사용한다.
- 최종 제목: `다음 예약부터, 고객정보까지 함께 남겨보세요`
- CTA: `14일 무료로 시작하기`
- 보조 문구: `카드 등록 없이 실제 매장 화면에서 확인할 수 있습니다.`

전환 역할: 남은 불안을 해소한 직후 가입 행동을 받는다.

## 5. 헤더와 모바일 전환 장치

- 헤더 메뉴: `어떻게 달라지나요`, `실제 화면`, `시간 가치`, `요금제`
- 우측 행동: `로그인`, `14일 무료 시작`
- 모바일에서는 메뉴를 숨기고 `14일 무료 시작`만 유지한다.
- 모바일 화면을 30% 이상 내리면 하단에 고정 CTA를 노출한다.
- 고정 CTA가 콘텐츠나 법적 링크를 가리지 않도록 푸터 진입 시 해제한다.

## 6. 디자인 방향

- 기본 색: 흰색, 잉크색, 옅은 회색
- 기능 상태에만 초록, 버건디, 파랑을 제한적으로 사용한다.
- 파란색 카드가 반복되는 현재 인상을 줄이고, 섹션은 전체 폭 밴드와 넓은 실제 화면 중심으로 구성한다.
- 카드 안에 카드를 넣지 않는다.
- 실제 UI 이미지는 흐리게 처리하거나 과도하게 자르지 않는다.
- 히어로와 기능 설명에 생성 이미지, 가짜 대시보드, 임의 목업을 사용하지 않는다.
- 브라우저·휴대폰 프레임은 실제 캡처를 담는 최소한의 CSS 프레임으로만 사용한다.
- 긴 설명보다 `제목 1개 + 결과 1개 + 실제 화면 1개`의 리듬을 유지한다.

2. 월간 캘린더

4. 보호자·반려동물 정보 입력 화면
5. 예약 접수 직후 오너 화면
6. 고객관리 목록과 고객 상세
7. 알림 설정과 알림톡 발송 현황
8. 직원별 일정 또는 담당자 필터

캡처 데이터는 실존 고객 개인정보를 사용하지 않고 데모 매장 데이터만 사용한다.

## 8. 전환 측정

- `landing_view`
- `hero_signup_click`
- `actual_screen_click`
- `pricing_plan_click` + 선택 플랜 코드
- `faq_open` + 질문 키
- `signup_start`
- `signup_complete`
- `trial_activated`

광고의 UTM 값은 가입 완료까지 유지한다. 첫 구현 후에는 방문 수보다 `랜딩 -> 가입 시작`, `가입 시작 -> 가입 완료`, `가입 완료 -> 체험 활성화` 전환율을 우선 본다.
