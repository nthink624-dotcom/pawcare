# PetManager Mobile UI Standard

## 1. 목적

이 문서는 펫매니저 모바일 오너 앱 및 관련 UI의 일관성을 유지하기 위한 UI 기준을 정의한다.

목표:
- 화면마다 다른 UI를 하나의 문법으로 통일
- 기능은 유지하면서 UI/UX만 정리
- 소상공인 오너가 빠르게 이해하고 사용할 수 있는 구조 유지

---

## 2. 전체 방향

- 따뜻하고 부드러운 인상 유지
- 실무형 관리자 SaaS 스타일 유지
- 과한 장식 제거
- 정보 우선순위 중심 설계
- 한 화면에 메인 행동은 1개만 강조
- "예쁨"보다 "이해 속도" 우선

---

## 3. 페이지 구조

모든 페이지는 기본적으로 아래 구조를 따른다.

1. Header
2. Content
3. Primary Action (또는 Secondary Links)

---

## 4. 헤더 규칙

구조:

- Back Button
- Page Title
- Description (Optional)

규칙:

- 뒤로가기 위치는 항상 동일
- 제목은 한눈에 보이도록
- 설명은 최대 2줄
- 아이콘은 필요한 경우만 사용

---

## 5. Typography

| Type | Size |
|------|------|
| Main Title | 28 |
| Page Title | 24 |
| Section Title | 18 |
| Card Title | 16 |
| Body | 14 |
| Caption | 12 |

규칙:

- 한 화면에서 4단계 이하만 사용
- 10px 이하 금지
- 작은 텍스트는 보조 정보에만 사용

---

## 6. Spacing

Scale:

- 8 / 12 / 16 / 20 / 24 / 32

규칙:

- 기본 padding: 16~20
- 카드 내부: 16 or 20
- 섹션 간 간격: 최소 24
- 임의 spacing 금지

---

## 7. Components

### Button

- Height: 48
- Primary: Filled
- Secondary: Outline / Light BG
- Text Button: Lowest priority

Rules:

- 메인 버튼 1개만 강조
- 3개 이상 동급 버튼 금지

---

### Input

- Height: 48
- Label 필수
- Placeholder는 보조 정보

---

### Card

사용:

- 섹션 구분
- 요약 정보
- 리스트 아이템

금지:

- 카드 안 카드
- 긴 텍스트 카드화

---

### Badge

- 상태 표현은 badge로 통일
- 동일 상태 = 동일 스타일

---

## 8. CTA 규칙

모든 화면은 아래 중 하나만 메인 행동으로 설정한다.

- 조회
- 다음 단계
- 저장
- 결제
- 예약
- 로그인

규칙:

- 메인 CTA 1개만 강조
- 나머지는 보조로 이동

---

## 9. 페이지별 규칙

### 관리자 홈
- "오늘 할 일" 중심
- 카드 최소화
- 시간 흐름 기반

### 고객관리
- 목록 = 조회
- 상세 = 수정
- 삭제 노출 최소화

### 예약조회
- 검색 → 결과 구조 유지

### 로그인/회원가입
- 메인 행동 1개
- 인증 단계 분리

### 결제/플랜
- 비교 단순화
- 카드 안 카드 금지

### 소비자 예약
- 단계형 유지
- 입력 vs 선택 분리

### 약관/사업자
- 텍스트 중심
- 카드 금지

---

## 10. 절대 금지

- 카드 안 카드 구조
- 3개 이상의 메인 버튼
- 타이포 혼합
- 임의 spacing
- 기능 흐름 변경

---

## 11. 핵심 원칙 요약

1. 빠르게 이해되어야 한다
2. 메인 행동은 하나
3. 구조로 정보 전달
4. 카드 절제
5. 같은 것은 항상 같게
