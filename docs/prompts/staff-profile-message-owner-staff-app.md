# 직원 프로필 메시지 연동 작업 프롬프트

펫매니저 오너/직원 앱에 `직원 프로필 메시지` 기능을 추가해주세요.

목표:
- 고객 예약 페이지 첫 화면의 직원 소개 영역은 `직원 이름 + 직원 프로필 메시지`를 보여줘야 합니다.
- 기존처럼 매장 소개 문구를 직원 이름 아래에 보여주면 안 됩니다.
- 오너 또는 직원이 직원관리 화면에서 각 직원의 프로필 메시지를 입력/수정할 수 있어야 합니다.

데이터 기준:
- DB 컬럼명은 `staff_members.profile_message`입니다.
- 프론트 타입 필드는 `profileMessage`로 사용합니다.
- 메시지는 고객에게 노출되는 짧은 한 줄 소개입니다.
- 권장 최대 길이는 80자, API 허용 최대 길이는 160자입니다.

UI 요구사항:
- 직원관리 상세/수정 화면에 `직원 프로필 메시지` 입력칸을 추가합니다.
- placeholder 예시는 `아이 성향에 맞춰 차분하게 미용해드려요.`를 사용합니다.
- 고객 예약 첫 화면에서는 직원의 `displayName`이 있으면 우선 사용하고, 없으면 `name`을 사용합니다.
- 직원 프로필 메시지가 비어 있으면 `아이 성향에 맞춰 차분하게 미용해드려요.`를 fallback으로 표시합니다.
- 직원 프로필 사진이 있으면 같이 사용하고, 없으면 기존 기본 아이콘/아바타를 유지합니다.
- 고객 예약 첫 화면의 직원 프로필 영역은 한 명만 고정 표시하지 말고, 좌우 터치 스와이프로 모든 직원의 사진, 이름, 프로필 메시지를 확인할 수 있게 만듭니다.
- 직원이 2명 이상이면 현재 위치를 알 수 있는 작은 dot indicator를 표시합니다.
- 직원 프로필 스와이프는 세로 스크롤을 방해하지 않도록 가로 이동 임계값을 둡니다.

동기화 요구사항:
- 오너 웹, 오너 앱, 직원 앱이 같은 `staff_members` 데이터를 사용해야 합니다.
- 직원관리에서 저장한 `profileMessage`는 고객 예약 페이지와 미리보기에 즉시 반영되어야 합니다.
- 기존 `displayName`, `profileImageUrl`, `titlePrefix`, `position`, `chipColorIndex` 저장 흐름은 깨지면 안 됩니다.

검증:
- 직원관리에서 프로필 메시지를 저장한다.
- 고객 예약 페이지 첫 화면에서 직원 이름 아래에 저장한 메시지가 보이는지 확인한다.
- 직원이 여러 명일 때 고객 예약 첫 화면에서 좌우 스와이프로 모든 직원 프로필을 넘겨볼 수 있는지 확인한다.
- 매장 소개를 바꿔도 직원 프로필 메시지 영역이 매장 소개로 바뀌지 않는지 확인한다.
- 직원 프로필 메시지가 비어 있을 때 fallback 문구가 보이는지 확인한다.
