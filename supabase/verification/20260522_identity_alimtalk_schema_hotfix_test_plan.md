# 2026-05-22 본인확인/알림톡 크레딧 스키마 보강

## Supabase SQL Editor 제목

`2026-05-22 본인확인/알림톡 크레딧 스키마 보강`

## 적용 순서

1. 개발 Supabase에 `20260522_identity_alimtalk_schema_hotfix.sql` 실행
2. 아래 스모크 테스트 통과 확인
3. 운영 Supabase에 같은 SQL 실행
4. 운영에서 같은 스모크 테스트를 짧게 재확인

## 개발 DB 적용 후 확인할 것

1. 회원가입 또는 비밀번호 재설정 화면에서 본인확인 창이 열려야 합니다.
2. 본인확인 완료 후 `Already verified`, `이미 완료된 본인인증 요청입니다`, `Not sent` 같은 오류로 흐름이 막히지 않아야 합니다.
3. 회원가입의 다음 단계로 넘어가거나 비밀번호 재설정 입력 영역이 열려야 합니다.
4. 우측 상단 카카오톡 아이콘 메뉴에서 알림톡 잔여 수량 조회가 에러 없이 열려야 합니다.
5. 알림톡 잔여 수량이 없으면 발송이 차단되고, 잔여 수량이 있으면 성공 발송 시 1건 차감되어야 합니다.
6. 발송 실패 시 차감된 1건이 환불 이벤트로 되돌아와야 합니다.

## 운영 DB 적용 후 확인할 것

1. 운영 사이트에서 로그인/회원가입 흐름이 기존처럼 열리는지 확인합니다.
2. 본인확인 재시도 시 같은 오류가 반복되지 않는지 확인합니다.
3. 오너 화면 우측 상단 카카오톡 아이콘 메뉴가 정상적으로 열리는지 확인합니다.
4. 알림톡 발송 테스트는 실제 차감이 발생할 수 있으므로 테스트 매장 또는 테스트 고객으로만 확인합니다.

## 주의

- 이 SQL은 기존 회원, 매장, 예약, 알림 데이터를 삭제하지 않습니다.
- 알림톡 크레딧 기본값은 매장별 0건입니다. 실제 무료/유료 수량 부여는 별도 정책/관리 화면에서 처리해야 합니다.
- 운영 적용 전에는 대상 프로젝트가 운영 Supabase인지 다시 확인해야 합니다.
- Supabase 적용 후에도 같은 에러가 보이면 PostgREST schema cache가 늦게 갱신된 경우일 수 있으니 30초 후 새로고침합니다.
