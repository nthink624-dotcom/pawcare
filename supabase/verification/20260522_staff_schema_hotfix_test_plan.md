## 적용 순서

1. 개발 Supabase에 `20260521_staff_schema_hotfix.sql` 실행
2. 아래 스모크 테스트 통과 확인
3. 운영 Supabase에 같은 SQL 실행
4. 운영에서 같은 스모크 테스트를 짧게 재확인

## 개발 DB 적용 후 확인할 것

1. 직원 관리 페이지 진입 시 `staff_members` 에러가 없어야 합니다.
2. 직원 목록에서 직원 1명을 추가하고 새로고침 후에도 남아 있어야 합니다.
3. 직원 이름, 역할, 출근/퇴근 시간 수정 후 새로고침해도 값이 유지되어야 합니다.
4. 직원 근무표에서 특정 날짜 근무/휴무/연차/반차를 저장하고 새로고침 후 유지되어야 합니다.
5. 스케줄 추가 모달에서 담당자를 선택해 예약을 등록했을 때 스케줄 보드에 해당 담당자 컬럼으로 보여야 합니다.
6. 기존 예약 카드의 담당자 또는 시간을 바꾼 뒤 새로고침해도 유지되어야 합니다.
7. 같은 직원에게 시간이 겹치는 예약을 만들려고 하면 저장이 막혀야 합니다.

## 운영 DB 적용 후 확인할 것

1. 운영 사이트에서 직원 관리 페이지가 열리는지 확인합니다.
2. 테스트 직원 1명을 추가했다가 필요하면 비활성화/삭제 처리합니다.
3. 스케줄 추가에서 담당자 선택 드롭다운이 정상 노출되는지 확인합니다.
4. 운영 고객 예약/스케줄 화면에 기존 예약이 사라지지 않았는지 확인합니다.
