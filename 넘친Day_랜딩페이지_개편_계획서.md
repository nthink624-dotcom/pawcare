# 넘친 Day 랜딩페이지 개편 계획서

## 1. 배경

petmanager.co.kr(넘친 Day) 랜딩페이지를 새로 디자인하고 카피를 전면 개편. 기존 페이지는 디자인이 다소 밋밋하고 명령조 카피("~하세요")로 구성되어 있었으며, 실 서비스에서 화면 캡처 이미지가 깨져 보이는 문제가 있었음.

## 2. 진단

- 화면 캡처 이미지(`actual-schedule.png` 등 7개) 자체는 프로젝트 폴더 내 정상 파일로 확인됨.
- 로고 파일명이 `넘친 Day.svg`처럼 한글+공백을 포함하고 있어, 배포 환경에서 유니코드 정규화 차이로 깨질 가능성이 높은 것으로 진단.
- 실 서비스에서 여전히 깨진다면 이미지가 아직 커밋/배포되지 않았을 가능성도 있음 — git 상태 및 배포 로그 확인 필요.

## 3. 핵심 전략 피벗: 시간 절약 → 손실 회피

초기에는 "하루 30분을 돌려드립니다"라는 시간 절약 프레임으로 설계했으나, 논의 과정에서 다음 인사이트로 방향을 수정함.

- 1인 미용사는 미용 중 손이 비어있지 않아 전화를 물리적으로 받을 수 없음. 이때 놓친 전화 한 통은 곧 놓친 예약이며, 이를 메꿔줄 직원이 없어 그대로 매출 손실로 이어짐.
- 다인 매장은 같은 문제가 발생해도 다른 직원이 대응할 여지가 있어 타격이 상대적으로 작음.
- 따라서 "시간을 아껴준다"보다 "놓치는 예약(매출)을 막아준다"는 손실 회피 프레임이 1인 샵과 다인 매장 모두에 더 설득력 있음.
- 숫자는 새로 만들지 않고 기존에 검증된 하루 30분 / 월 15시간 / 월 154,800원(2026년 최저시급 10,320원 기준)을 "아낀 시간의 가치"가 아니라 "놓치지 않은 매출"로 재해석함.

## 4. 슬로건

**확정: 예약이 넘쳐도, 놓치는 손님은 없게**
브랜드명 '넘친 Day'와 직접 연결되고, 노쇼·매출 손실이라는 사장님의 가장 큰 두려움을 짚는다.

검토했던 다른 후보:
- 넘치던 하루에, 순서가 생깁니다 — 부드럽지만 구체적 이득이 약함.
- 예약 받는 동안, 손님을 기다리게 하지 마세요 — 고객 경험 관점이라 오너 소구력이 약함.
- 반려동물 미용샵 운영, 하루 30분을 돌려드립니다 — 숫자 중심이나 손실 회피보다 소구력이 약하다고 판단해 보류.

## 5. 정보 구조 (섹션 순서)

1. 헤더 (로고 + 내비게이션 + CTA)
2. 히어로 — 슬로건 + 헤드라인(손실 회피) + 절약 지표 3개 + 실제 화면 캡처
3. 공감 섹션(신규) — 사장님이 겪는 불편 4가지 카드
4. 실제 제품 화면 캡처 — 예약관리 / 캘린더 / 고객관리 / 예약 링크
5. 손실 방지 효과(숫자) — 30분/15시간/154,800원
6. 고객 예약 흐름 — 고객 모바일 화면 2장 + 셀링 포인트
7. 오너 화면 가치 (다크 섹션) — 예약관리/고객DB/알림톡
8. 요금제 3단
9. 최종 CTA

## 6. 확정 카피 (현재 코드에 반영됨)

**히어로**
- 태그라인: 예약이 넘쳐도, 놓치는 손님은 없게
- 헤드라인: 전화를 못 받아도, 예약은 놓치지 않습니다
- 서브카피: 미용하는 동안 걸려온 전화, 다시 여쭤보는 연락처, 깜빡한 안내 문자 — 놓치는 순간은 그대로 놓치는 매출이 됩니다. 예약, 보호자·반려동물 정보, 알림톡, 캘린더를 오너 화면 하나로 정리해서 손이 비어있지 않을 때도 예약이 끊기지 않게 합니다.

**공감 섹션**
- 안내 문구: 미용샵 사장님이라면 익숙한 하루
- 제목: 미용은 내가 제일 잘하는데, 그 사이 놓치는 손님이 생깁니다
- 설명: 예약을 많이 받는 날일수록 이런 순간이 늘어납니다. 넘친 Day는 이 네 가지부터 막아드립니다.
- 카드 4개: 예약 전화 받다가 손질이 늦어짐 / 연락처를 올 때마다 다시 여쭤봄 / 안내 문자를 깜빡해 노쇼 발생 / 지난 방문 기록을 찾느라 기다리게 함

**손실 방지 효과**
- 안내 문구: 숫자로 보는 손실 방지 효과
- 제목: 하루 30분이면 월 154,800원, 놓치지 않는 셈입니다
- 설명: 2026년 최저시급 10,320원 기준, 하루 30분이면 한 달에 15시간입니다. 놓친 전화, 노쇼, 반복 확인에 쓰던 시간을 매출로 환산하면 이렇습니다.

**최종 CTA**
- 안내 문구: 이제, 놓치던 예약을 잡을 시간
- 제목: 예약 받는 일이 곧 고객관리가 되도록

## 7. 추가 논의 사항 (미반영, 검토 필요)

- 고객 편의 메시지 보강: 현재는 오너 관점 설명이 중심. "고객도 앱 설치 없이 몇 번의 터치로 예약 완료"라는 고객 편의 포인트를 마지막 CTA 직전에 3-아이콘 요약 블록(고객 편의 / 오너 시간 / 매장 안전)으로 추가하는 안 검토 중.
- 문제-해결-숫자 매핑: 공감 섹션 카드 4개에 시간 배분(예: 전화 응대 10분, 연락처 재확인 2~3분 등)을 붙이는 안 — 실제 체감 데이터가 없어 임의 추정치이므로, 실제 데이터 확보 전까지는 보류 권장.
- A/B 테스트: Vercel Flags SDK, PostHog, GrowthBook 등으로 가능. 다만 트래픽이 적으면 통계적으로 유의미한 결과까지 시간이 걸림 — 방문자 규모 확인 후 방식 결정 필요.
